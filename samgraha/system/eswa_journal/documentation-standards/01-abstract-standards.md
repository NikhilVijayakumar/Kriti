## Purpose
The Abstract is a 150-250 word summary of the entire paper following the IMRaD structure. It must secure editor interest immediately.

## Application Context
**Template:**
```markdown
[1 sentence describing the real-world domain and importance]
```

## Research Gap
**Template:**
```markdown
[1 sentence explicitly stating what is currently missing or failing in this domain]
```

## Proposed Solution
**Template:**
```markdown
[1-2 sentences describing the novel framework/system proposed to address the gap]
```

## Experimental Setup
**Template:**
```markdown
[1 sentence mentioning the dataset or environment used for evaluation]
```

## Quantitative Results
**Template:**
```markdown
[1-2 sentences stating the EXACT numerical improvements achieved (e.g., +8.4% F1) against baselines]
```
**Audit Rule:** Must contain specific numbers/percentages. Claims like "high accuracy" are rejected.

## Practical Relevance
**Template:**
```markdown
[1 sentence concluding how this advances the state-of-the-art in applied contexts]
```

## Writing Guidance
- **Length:** 150-250 words maximum.
- **Do:** Quantify results, mention evaluation method.
- **Don't:** Add citations, mention specific tools/code, exceed word limit.
