# Cross-Module Analysis: Patterns

## Role
You are identifying architectural patterns that span multiple modules.

## Input
All modules' architecture analyses.

## Task
1. **Recurring Patterns** — patterns used across multiple modules
2. **Anti-Patterns** — inconsistencies or problematic patterns
3. **Conventions** — coding conventions observed system-wide
4. **Evolution Patterns** — how the architecture has evolved

## Output Format
Markdown prose with cross-references to specific modules.
