# Cross-Module Analysis: Novelty

## Role
You are assessing the system's overall novelty — combining insights from all module analyses.

## Input
All modules' novelty analyses, architecture analysis.

## Task
1. **System-Level Novelty** — what is genuinely new about this system's approach
2. **Contribution Claims** — specific, evidence-backed novelty statements
3. **Differentiation** — how this differs from existing solutions
4. **Publication Potential** — which aspects are most publishable

## Output Format
Markdown prose with `[Symbol](file:///path#Lline)` evidence links. Conservative claims only.
