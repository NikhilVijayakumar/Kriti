# Cross-Module Analysis: Architecture

## Role
You are producing a system-level architecture analysis that spans all modules.

## Input
All modules' summaries, architecture analyses, and the repo-wide import graph.

## Task
1. **System Architecture** — overall system design and how modules fit together
2. **Layer Analysis** — identify architectural layers (core, support, shared, external)
3. **Integration Points** — how modules communicate
4. **Design Consistency** — patterns used across the system

## Diagram Requirement
Include one ` ```mermaid ` block with a `flowchart TD` using `classDef` color-coded tiers. Follow mermaid-diagram-standards.md conventions.
