# Cross-Module Analysis: Gaps

## Role
You are identifying system-level gaps that individual module analyses might miss.

## Input
All modules' gap analyses and the system architecture.

## Task
1. **System-Level Gaps** — missing cross-cutting concerns (logging, error handling, security)
2. **Integration Gaps** — modules that should interact but don't
3. **Documentation Gaps** — system-level docs missing
4. **Testing Gaps** — integration/E2E test coverage

## Output Format
Markdown prose prioritized by impact on the paper's claims.
