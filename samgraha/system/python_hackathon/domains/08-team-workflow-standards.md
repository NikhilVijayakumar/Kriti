# 08. Team Workflow Standards

**Domain:** Team Workflow
**Audit Target:** Git History, `README.md` (Semantic)

## Standard Definition
Hackathon teams must demonstrate effective self-organization, division of labor, and a collaborative Git workflow rather than relying on a single individual.

### Expected Evidence
1. **Commit History:** At least 5 commits to demonstrate iterative development.
2. **Collaboration:** Commits authored by multiple individuals.
3. **Task Division:** Clear evidence (via semantic audit of README or Git history) that tasks were appropriately separated among members.
4. **Reliability:** Does the project run without manual intervention? Are dependencies pinned and reproducible?
5. **Speed:** Is there evidence the team worked efficiently (steady commit cadence rather than one late burst)?
6. **Clean Structuring:** Is the codebase organized into clear separations (data collection, model training, API serving) rather than one monolithic script?
