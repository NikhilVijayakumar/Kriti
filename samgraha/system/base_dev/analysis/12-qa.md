# QA Analysis Prompt

Guides the semantic-analysis step for the QA domain — read by whichever
agent runs it via MCP, on-demand, after `analyze_trend.py` has produced
`12-qa-trend.json`.

## Version
1.0.0

## Analysis Intent
QA's score moving is only informative if the reader knows *why*. This
step turns scores + trend + findings into: a short narrative, chart
selection, and fix-plan decisions.

## Inputs
- `12-qa-scores.json` (from `calculate.py`)
- `12-qa-trend.json` (from `analyze_trend.py`)
- `12-qa-validation.json`, `12-qa-det-eval.json`,
  `12-qa-sem-eval.json` (findings)

## Narrative Guidance
QA's audit rubric scores: Test Strategy priority reflected in test type
depth (C1), Integration Testing boundaries consistent with Architecture
(C2), and E2E journeys traceable to Design workflows (C3). Script
evidence from `unit-test-coverage` grounds factual claims.

The narrative should say:
- Is the score trend improving/declining, and by how much.
- If declining or flat: which of C1/C2/C3 is driving it.
- If C1 (priority-depth alignment) is failing, call out which test
  type sections don't match the stated priority — e.g. "high priority"
  but thin Integration Testing section.
- If C2 (Architecture cross-reference) is failing, name which
  integration boundaries don't match `05-architecture`.
- If `unit-test-coverage` script found coverage gaps, name them.

Two sections minimum: `Summary` and `Key Risks`.

## Visualization Guidance
- `domain_scores` — always include.
- `section_heatmap` — include only if 2+ failing sections.
- `rule_weights_heatmap` — include only if failing rules have
  `weight >= 1.5`.
- `trend_history` — include once `runs_compared >= 3`.

## Fix-Plan Trigger Criteria
- **Always trigger**: mandatory criterion failures (C1, C2).
- **Trigger if recurring**: C3 failing for 2+ consecutive runs.
- **Never trigger**: `info`-severity findings, or findings already
  covered by an open fix plan.

## Output Schema
```json
{
  "domain": "12-qa",
  "sections": [
    {"heading": "Summary", "text": "..."},
    {"heading": "Key Risks", "text": "..."}
  ]
}
```

```json
{
  "domain": "12-qa",
  "charts": ["domain_scores", "section_heatmap"],
  "focus_sections": ["test-strategy", "integration-testing"],
  "reason": "..."
}
```
