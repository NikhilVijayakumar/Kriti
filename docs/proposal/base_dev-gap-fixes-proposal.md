## 2. Root Cause Analysis

### GAP-001 — Stale Generated Files

**Not a code bug.** The 32 `plan/usecase/**/02-audit.md` files are mechanically
generated by `script/render_usecase_docs.py`. The current `generate_tier_audit_md()`
function (lines 146-220) already produces the correct format with executor scripts named:

```python
"2. **Evaluate rules** (`scripts/evaluate_rules.py`) — evaluate deterministic rules",
"3. **Evaluate semantic** (`scripts/evaluate_semantic.py`) — heuristic evaluation",
```

But the on-disk files still use an older format from before this function was updated:

```
1. Deterministic document audit: Run `audit/deterministic/document/{domain}.yaml` against the document.
```

This older format names the YAML data file, not the Python script that executes it.

**Fix:** Re-run `render_usecase_docs.py` to regenerate all 32 files.

### GAP-004 — Missing Pipeline Documentation

`plan/core/README.md` (32 lines) describes the loop conceptually but never lists
the actual scripts or their data flow. Meanwhile, `script/init.py` (902 lines) is a
fully-built orchestrator covering the entire pipeline (scaffold → structural →
content → validate → calculate → report → fix) — and 10 sibling scripts exist
that it calls. None of these appear in `plan/core/` documentation.

`grep -rl "init.py\|scaffold.py\|validate.py\|analyze.py" plan/` returns zero
matches — confirming the gap.

**Fix:** Expand `plan/core/README.md` with a Script Chain section.

### Change 2: Re-run `render_usecase_docs.py`

**Command:**

```bash
python samgraha/system/base_dev/script/render_usecase_docs.py \
  --system-root samgraha/system/base_dev
```

**Effect:** Regenerates 128 files (4 files × 4 use cases × 8 tiers):
- `01-generation.md` — Path A generation pipeline (already has script names)
- `02-audit.md` — Path B audit pipeline (**GAP-001 fix**: gains executor script names)
- `03-fix.md` — Fix loop details
- `04-analyze.md` — Analysis and fix-plan confirmation

**Before (02-audit.md current on-disk format):**

```markdown
1. Deterministic document audit: Run `audit/deterministic/document/{domain}.yaml` against the document.
2. Deterministic section audit: Run `audit/deterministic/section/{domain}/*.yaml` against each section.
3. Semantic document audit: Run `audit/semantic/document/{domain}.md` against the whole document.
4. Semantic section audit: Run `audit/semantic/section/{domain}/*.md` against each section.
```

**After (02-audit.md regenerated format):**

```markdown
Each domain in this tier follows the Path B pipeline (no scaffold/content phase):

1. **Pre-hook: staleness check** — skip if doc unchanged since last audit
2. **Evaluate rules** (`scripts/evaluate_rules.py`) — evaluate deterministic rules against existing docs
3. **Evaluate semantic** (`scripts/evaluate_semantic.py`) — heuristic semantic criteria evaluation
   - Pre-script: `scripts/gather_semantic_context.py` — gather check metrics as grounding evidence
4. **Validate** (`scripts/validate.py`) — run 18 deterministic check scripts
5. **Calculate** (`scripts/calculate.py`) — compute 4-bucket score from evaluated results
6. **Report** (`scripts/report.py`) — render markdown report
7. **Analyze** (`scripts/analyze.py`) — generate structured fix plan
8. **Visualize** (`scripts/visualize.py`) — generate 8 PNG charts
9. **Report HTML** (`scripts/report_html.py`) — render self-contained HTML report
10. **Fix** (semantic, conditional) — only if score < threshold; modify existing sections
```
